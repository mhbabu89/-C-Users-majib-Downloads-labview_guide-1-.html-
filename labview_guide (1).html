📍 How to Add Network Device

In NI-MAX: Devices and Interfaces → Network Devices → Add Network Device → VISA TCP/IP Resource → Manual Entry

Enter IP: 192.168.0.2 and Port: 1394, select Raw Socket

⚠️ If IDN? fails

On the K6221 front panel:
Press COMM → select LAN → verify Status = Enabled

Also check that your PC and K6221 are on the same subnet (192.168.0.x)

2
Add Functions to check_run_2.py
LabVIEW Python Node needs callable functions — add these at the bottom of your file
⚠️ Don't delete anything from your original script

Just add the code below to the bottom of check_run_2.py. Keep all your existing code exactly as it is.

# ───────────────────────────────────────────── # ADD THESE FUNCTIONS AT THE BOTTOM OF check_run_2.py # ───────────────────────────────────────────── def run_waveform(amp_mA=1.0, comp_V=2.52, freq_Hz=7.0, wave="SIN", marker_enable=False, marker_angle=0.0): """Called directly from LabVIEW Python Node""" amp_A = float(amp_mA) * 1e-3 # mA → A ← IMPORTANT! import pyvisa rm = pyvisa.ResourceManager("@ivi") inst = None try: inst = rm.open_resource(RESOURCE) inst.timeout = 10000 inst.read_termination = "\n" inst.write_termination = "\n" idn = inst.query("*IDN?") safe_stop(inst) inst.write("SOUR:FUNC CURR") inst.write("SOUR:CURR:COMP {:.6f}".format(float(comp_V))) inst.write("SOUR:WAVE:FUNC {}".format(wave)) inst.write("SOUR:WAVE:FREQ {:.6f}".format(float(freq_Hz))) inst.write("SOUR:WAVE:AMPL {:.12f}".format(amp_A)) inst.write("SOUR:WAVE:OFFS 0") if marker_enable: inst.write("SOUR:WAVE:PMAR:STAT ON") inst.write("SOUR:WAVE:PMAR:LEV {:.2f}".format(float(marker_angle))) else: inst.write("SOUR:WAVE:PMAR:STAT OFF") inst.write("OUTP ON") inst.write("SOUR:WAVE:ARM") inst.write("SOUR:WAVE:INIT") return "OK: {} {}Hz {}mA IDN={}".format(wave, freq_Hz, amp_mA, idn.strip()) except Exception as e: return "ERROR: {}".format(str(e)) finally: if inst: inst.close() rm.close() def stop_waveform(): """Called from LabVIEW STOP button""" import pyvisa rm = pyvisa.ResourceManager("@ivi") try: inst = rm.open_resource(RESOURCE) inst.timeout = 5000 inst.read_termination = "\n" inst.write_termination = "\n" safe_stop(inst) inst.close() return "Stopped OK" except Exception as e: return "Stop ERROR: {}".format(str(e)) finally: rm.close()
✅ Save the file then test it

Open Command Prompt and run:
python -c "import check_run_2; print('OK')"

If it prints OK with no errors, the file is ready for LabVIEW.

Troubleshooting Checklist

Python version matches LabVIEW bitness (both 32-bit or both 64-bit)
CRITICAL
NI-MAX shows K6221 connected, IDN? returns response
NETWORK
Module path in Python Node is full absolute path (e.g. C:\Users\...\check_run_2.py)
PATH
Function name is exactly run_waveform (case sensitive)
NAME
pyvisa is installed in the same Python that LabVIEW is using

Status indicator shows "OK: ..." (not "ERROR: ...")
TEST
❌ "LabVIEW: Python version mismatch"

Check LabVIEW: Help → About LabVIEW — is it 32-bit or 64-bit?
Then install the matching bitness Python. They MUST match.

❌ "Module not found" error

Use the full path with double backslashes:
C:\\Users\\Name\\Desktop\\check_run_2.py

❌ Status shows "ERROR: VISA error"

K6221 not reachable. Re-run NI-MAX test first. Check K6221 COMM → LAN settings on the instrument front panel.

✅ Status shows "OK: SIN 7.0Hz..."

Everything is working! The K6221 is outputting the waveform. Press STOP to safely stop the output when done.
